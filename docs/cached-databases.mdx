## What it is

Cached Databases are pre-generated datasets created based on popular or frequently repeated search patterns from our users. These datasets are refreshed automatically, which allows you to receive results instantly and at a lower cost compared to running a live search.

## When to use it

- when you need a large list of contacts or companies quickly, without waiting
- when your query is standard and doesn't require unique or complex filters
- when you want to reduce costs, as cached datasets are cheaper than Live Search

## How it works

- When you run an ICP search, Generect first checks whether relevant cached data already exists.
- If cached data is available, it is returned immediately. This is faster and more cost-efficient because the dataset was generated earlier.
- If the returned data is not sufficient or you need fresher results (for example, updated today rather than a week ago), you can run Live Search, which generates up-to-date results in real time for an additional cost.
- If no cached data exists for the ICP you specified, the system automatically switches to Live Search and fetches new real-time results.

<Tip>
  Cached databases are perfect for large-scale workflows where data freshness is less critical than speed and cost efficiency
</Tip>
